  5  """ Plugin architecture. Contains the base for writing new plugins and 
  6  facilities managing it. 
  7   
  8  Singletons 
  9  ========== 
 10  - RootPluginSingleton: root plugin for the media server, initial target of any 
 11                         browse. All plugin folders are added inside this root 
 12                         plugin. 
 13  - PluginManagerSingleton: takes care of automatically finding the plugins (on 
 14                            the plugin folder), instantiating them and loading 
 15                            them asynchronously using PluginLoader. 
 16   
 17  Basic use 
 18  ========= 
 19  The basic use of the plugin manager is to set some class of yours as a listener 
 20  and load the plugins. Note that your class MUST have both on_start_plugin() and 
 21  on_end_plugin() methods implemented. 
 22   
 23  >>> from brisa.services.cds.plugin import PluginManagerSingleton 
 24  >>> someclass = SomeClass() 
 25  >>> PluginManagerSingleton.listener = someclass 
 26  >>> PluginManagerSingleton.load_plugins() 
 27   
 28  Among other uses you might not want to set up a listener and only load the 
 29  plugins. Setting a listener is completely optional. 
 30  """
