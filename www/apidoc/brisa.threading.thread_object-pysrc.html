 23   
 24      When a ThreadObject is instantiated, it registers itself on the

 26   
 27      Since ThreadObject is a Thread, all the logic of your Thread should be 
 28      written in the run() method, which does nothing by default. 
 29

 31      routine, you should do this on the prepare_to_start() method. 
 32   
 33      Analogous to the prepare_to_start() method, ThreadObject's also provides 
 34      a prepare_to_stop() method, in which you should write actions to be done 
 35      before exiting. 
 36   
 37      ThreadObjects are named with the template "ThreadObject %d: %s" where 
 38      %d is the unique ThreadObject number and %s is your class that inherits 
 39      from ThreadObject. This keeps debugging easy. 
 40   
 41      Note that if you overwrite stop() or start() methods, both prepare_to_start 
 42      and prepare_to_stop methods will become useless, since the logic for these 
 43      calls will be lost (unless you copy it, which is not a very smart thing).

 46      a way that is_running() method is often read and handles the exiting of
